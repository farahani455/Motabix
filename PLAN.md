# MotabixCode - Project Plan

## What This Project Is
A Visual Studio Code extension called **Motabix** that provides a chat-style sidebar panel (like GitHub Copilot Chat) on the right side of VS Code.

## Current Status
The extension is working and has a clean folder structure.

## Current Folder Structure
```

  commands/
```

## What Comes Next ­čöť

### Step 1: API Call Ôťů
- Connect the webview send button to a real API
- Options: OpenAI API, custom Node.js backend, or another AI service
- Flow:
  1. Ôťů User types message and clicks Send
  2. Ôťů Webview sends message to extension via `vscode.postMessage`
  3. Ôťů Extension receives message in `onDidReceiveMessage`
  4. Ôťů Extension makes HTTP request to API
  5. Ôťů Extension sends response back to webview via `webviewView.webview.postMessage`
  6. Ôťů Webview displays the response
- Recommended package: `axios` or built-in `https` module

## Key Concepts Learned
- VS Code extension anatomy: `package.json` + `extension.ts`
- `TreeDataProvider` vs `WebviewViewProvider`
- `viewsContainers` + `views` in `package.json`
- `workbench.action.toggleAuxiliaryBar` to open Secondary Side Bar
- TypeScript: `export function`, `export class`, `implements`, types, interfaces
- Module pattern: one responsibility per file
- `export` vs `module.exports` ÔÇö always use `export` in TypeScript
- TypeScript compiles to JS ÔÇö Node.js runs the compiled JS, not `.ts` directly
